# ROADMAP

# Jour 1 — Mise en route
- Env: setup + repo + CI de base
- Clarifier le scope MVP (features essentielles seulement)
- Structure du projet posée

Livrable: projet installable, CI qui tourne, scope MVP écrit


# Jour 2 — MVP jouable
- Coder les features cœur (priorisées la veille)
- Tests unitaires sur ces features
- Petite démo interne pour valider le gameplay/UX

Livrable: MVP minimal jouable validé en démo

# Jour 3 — Durcir le jeu
- Tests d'intégration (parcours complets)
- Correction des bugs visibles en démo
- Ajustements perf simples (pas de refonte)

Livrable: parcours intégré qui passe sans crash, liste des bugs corrigés


# Jour 4 — Prépa démo finale
- Packager la build (staging/devenv propre)
- Passage de la checklist: tests verts, assets ok, son/UX ok
- Dry-run de la démo finale

Livrable: build prête à montrer + checklist validée

# Dernier jour — Show & tell
- Test de régression rapide sur le périmètre MVP
- Slides/notes de démo + mini guide d’usage
- Présentation

Livrable: démo fluide, app utilisable, doc courte
